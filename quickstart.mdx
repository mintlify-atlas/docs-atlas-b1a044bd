Get TREK up and running on your machine with a single Docker command.

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) installed and running

## Run TREK

<Steps>
  <Step title="Start the container">
    Generate an encryption key and start TREK in one command:

    ```bash
    ENCRYPTION_KEY=$(openssl rand -hex 32) docker run -d -p 3000:3000 \
      -e ENCRYPTION_KEY=$ENCRYPTION_KEY \
      -v ./data:/app/data -v ./uploads:/app/uploads mauriceboe/trek
    ```

    This pulls the `mauriceboe/trek` image, starts TREK on port `3000`, and persists your data in `./data` and `./uploads`.
  </Step>
  <Step title="Open your browser">
    Navigate to [http://localhost:3000](http://localhost:3000). The TREK login screen will appear once the container is ready — this takes about 15 seconds on first start.
  </Step>
  <Step title="Register the first admin">
    Click **Register** and create your account. The first user to register on a fresh TREK instance is automatically granted admin access.

    <Note>
      There is no separate admin setup step. Whoever registers first becomes the admin. If you are self-hosting for a group, register before sharing the URL.
    </Note>
  </Step>
  <Step title="Create your first trip">
    After logging in, click **New Trip** on the dashboard. Set a destination, date range, and title — then start adding places, inviting collaborators, and building your itinerary.
  </Step>
</Steps>
