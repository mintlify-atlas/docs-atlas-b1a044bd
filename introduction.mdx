## What is TREK?

TREK gives you a full-featured travel planner that you host yourself. Every trip lives in your SQLite database, every file in your own storage — no third-party cloud required.

**Plan trips together.** Invite collaborators and plan in real time via WebSocket. Changes appear instantly on every connected device.

**Bring your own maps.** Use OpenStreetMap for free, or plug in a Google Places API key for photo-rich place search with ratings and opening hours.

**Track everything.** Budgets with multi-currency support and per-person splitting, packing lists with reusable templates, reservations for flights, hotels, and restaurants — all in one place.

**Works like a native app.** Install TREK as a PWA on iOS and Android directly from the browser. Offline support via Workbox keeps your plans accessible without internet.

## Get started in 3 steps

<Steps>
  <Step title="Run TREK with Docker">
    Start the server with a single command — your data is persisted in local volumes.

    ```bash
    ENCRYPTION_KEY=$(openssl rand -hex 32) docker run -d -p 3000:3000 \
      -e ENCRYPTION_KEY=$ENCRYPTION_KEY \
      -v ./data:/app/data -v ./uploads:/app/uploads mauriceboe/trek
    ```
  </Step>
  <Step title="Register the first user">
    Open `http://localhost:3000` in your browser. The first user to register automatically becomes the admin.
  </Step>
  <Step title="Create your first trip">
    Click **New Trip**, set a destination and date range, then invite collaborators and start planning.
  </Step>
</Steps>

<Note>
  For production use, deploy with Docker Compose behind a reverse proxy with HTTPS. See the [Installation guide](/installation) for a full production setup.
</Note>
